Problem

Users currently only have email/password authentication. This creates friction during onboarding and lacks the security benefits of enterprise-grade SSO. We need to implement OAuth2 to support Google, GitHub, and Microsoft login providers.

Requirements

  • Integration with Passport.js strategy for OAuth2.
  • Automated user account linking based on verified email addresses.
  • Secure token storage using encrypted sessions.
  • Support for PKCE flow for mobile/desktop client security.
