Navigate giữa node frames trong 1 Design?

Sau khi load 1 Design có nhiều frames, user sẽ duyệt chúng như thế nào

A

Sidebar list + Canvas đơn

Panel trái liệt kê tất cả node frames dạng thumbnail/list. Click 1 frame → canvas giữa hiển thị frame đó + panel phải hiện annotation cho frame đang chọn.

┌─────────┬──────────────────┬──────────┐
│ Designs │ │ Notes │
│ ├ Frame1│ Canvas: │ [tags] │
│ ├ Frame2│ Frame1 image │ text... │
│ └ Frame3│ │ │
└─────────┴──────────────────┴──────────┘

Pros

  • Quen thuộc — giống Figma layers panel
  • Canvas lớn, xem chi tiết từng frame

Cons

  • Chỉ thấy 1 frame tại 1 thời điểm
B

Grid overview + Detail view

Mặc định hiển thị grid tất cả frames (thumbnails). Click 1 frame → zoom vào detail view (canvas + layers + annotation). Có nút back để quay lại grid.

Grid view:              Detail view:
┌──────────────────┐   ┌─────┬──────────┬──────┐
│ [img1] [img2] │ → │Layer│ Canvas │Notes │
│ [img3] [img4] │   │ │ Frame1 │ │
└──────────────────┘   └─────┴──────────┴──────┘

Pros

  • Tổng quan tốt — thấy hết frames cùng lúc
  • Dễ so sánh visual giữa các frames

Cons

  • Thêm 1 bước navigate (grid → detail → back)
C

Kết hợp: Sidebar tree + Grid/Detail toggle (Recommended ⭐)

Sidebar trái hiện tree: Designs → Node frames. Panel giữa có toggle giữa Grid view (overview tất cả frames) và Detail view (canvas + layers cho 1 frame). Panel phải: annotations cho frame đang active.

┌──────────┬──────────────────┬──────────┐
│ Designs │ [Grid] [Detail] │ Annotate │
│ ▼ Design1│ │ │
│   Frame1 │ (grid or canvas) │ notes │
│   Frame2 │ │ [tags] │
│ ▼ Design2│ │ │
└──────────┴──────────────────┴──────────┘

Pros

  • Best of both — overview khi cần, detail khi annotate
  • Tree sidebar cho phép navigate nhanh giữa designs + frames
  • Không mất context khi switch view

Cons

  • Phức tạp nhất để implement
